How we work

IEC 62304-aligned development process

01
IEC §5.1
Software Development Planning
We establish the development plan before a single line of code is written. Safety classification, tool qualification, and roles are locked before kickoff.
SDP Safety Classification Tool Qualification
02
IEC §5.2
Requirements Analysis
Stakeholder and system requirements translated into a traceable Software Requirements Specification with usability engineering inputs.
SRS Usability Inputs
03
IEC §5.3
Architectural Design
System architecture designed with safety and security in mind. SOUP items identified and documented with known anomalies and required functionality.
SADD SOUP List
04
IEC §5.4–5.5
Detailed Design & Implementation
Fully documented detailed design followed by implementation with peer-reviewed, standards-conformant code. Unit tests written and executed per plan.
Source Code Unit Test Results Code Review Records
05
ISO 14971
Risk Management
Hazard analysis, risk estimation, and control measures documented throughout the lifecycle — not added at the end. Residual risk evaluated per acceptability criteria.
Risk Management File (RMF) Hazard Analysis Risk Controls
06
IEC §5.6–5.7
Verification & Validation
Integration and system testing executed against approved protocols. Requirements traceability matrix closed. Anomalies resolved or risk-accepted per process.
Integration Test Protocols & Reports System Test Protocols & Reports RTM
07
IEC §5.8
Configuration Management & Release
Release baseline locked with full SBOM. All configuration items under version control with immutable release records and cybersecurity documentation.
SBOM Configuration Management Plan Release Records Cybersecurity Docs
08
IEC §6
Post-Market Maintenance
Ongoing monitoring, problem resolution, and change management per your Software Maintenance Plan. We stay engaged so you're always compliant post-launch.
Software Maintenance Plan Problem Resolution Records Change Requests
